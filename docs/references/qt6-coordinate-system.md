# Coordinate System

The coordinate system is controlled by the [QPainter](qpainter.html) class. Together with the [QPaintDevice](qpaintdevice.html) and [QPaintEngine](qpaintengine.html) classes, [QPainter](qpainter.html) form the basis of Qt\'s painting system, Arthur. [QPainter](qpainter.html) is used to perform drawing operations, [QPaintDevice](qpaintdevice.html) is an abstraction of a two-dimensional space that can be painted on using a [QPainter](qpainter.html), and [QPaintEngine](qpaintengine.html) provides the interface that the painter uses to draw onto different types of devices.

The [QPaintDevice](qpaintdevice.html) class is the base class of objects that can be painted: Its drawing capabilities are inherited by the [QWidget](qwidget.html), [QImage](qimage.html), [QPixmap](qpixmap.html), [QPicture](qpicture.html), and [QOpenGLPaintDevice](qopenglpaintdevice.html) classes. The default coordinate system of a paint device has its origin at the top-left corner. The *x* values increase to the right and the *y* values increase downwards. The default unit is one pixel on pixel-based devices and one point (1/72 of an inch) on printers.

The mapping of the logical [QPainter](qpainter.html) coordinates to the physical [QPaintDevice](qpaintdevice.html) coordinates are handled by [QPainter](qpainter.html)\'s transformation matrix, viewport and \"window\". The logical and physical coordinate systems coincide by default. [QPainter](qpainter.html) also supports coordinate transformations (e.g. rotation and scaling).

### Aliased Painting

When drawing, the pixel rendering is controlled by the [QPainter::Antialiasing](qpainter.html#RenderHint-enum) render hint.

The [RenderHint](qpainter.html#RenderHint-enum) enum is used to specify flags to [QPainter](qpainter.html) that may or may not be respected by any given engine. The [QPainter::Antialiasing](qpainter.html#RenderHint-enum) value indicates that the engine should antialias edges of primitives if possible, i.e. smoothing the edges by using different color intensities.

But by default the painter is *aliased* and other rules apply: When rendering with a one pixel wide pen the pixels will be rendered to the *right and below the mathematically defined points*. For example:

Note that for historical reasons the return value of the [QRect::right](qrect.html#right)() and [QRect::bottom](qrect.html#bottom)() functions deviate from the true bottom-right corner of the rectangle.

[QRect](qrect.html)\'s [right](qrect.html#right)() function returns [left](qrect.html#left)() + [width](qrect.html#width)() - 1 and the [bottom](qrect.html#bottom)() function returns [top](qrect.html#top)() + [height](qrect.html#height)() - 1. The bottom-right green point in the diagrams shows the return coordinates of these functions.

We recommend that you simply use [QRectF](qrectf.html) instead: The [QRectF](qrectf.html) class defines a rectangle in the plane using floating point coordinates for accuracy ([QRect](qrect.html) uses integer coordinates), and the [QRectF::right](qrectf.html#right)() and [QRectF::bottom](qrectf.html#bottom)() functions *do* return the true bottom-right corner.

Alternatively, using [QRect](qrect.html), apply [x](qrect.html#x)() + [width](qrect.html#width)() and [y](qrect.html#y)() + [height](qrect.html#height)() to find the bottom-right corner, and avoid the [right](qrect.html#right)() and [bottom](qrect.html#bottom)() functions.

## Transformations

By default, the [QPainter](qpainter.html) operates on the associated device\'s own coordinate system, but it also has complete support for affine coordinate transformations.

You can scale the coordinate system by a given offset using the [QPainter::scale](qpainter.html#scale)() function, you can rotate it clockwise using the [QPainter::rotate](qpainter.html#rotate)() function and you can translate it (i.e. adding a given offset to the points) using the [QPainter::translate](qpainter.html#translate)() function.

::: table
  -------------------------------- -------------------------------------------------- ------------------------------------------------ --------------------------------------------------------
  ![](images/qpainter-clock.png)   ![](images/qpainter-rotation.png)                  ![](images/qpainter-scale.png)                   ![](images/qpainter-translation.png)
  nop                              [rotate](qpainter.html#rotate)()   [scale](qpainter.html#scale)()   [translate](qpainter.html#translate)()
  -------------------------------- -------------------------------------------------- ------------------------------------------------ --------------------------------------------------------

You can also twist the coordinate system around the origin using the [QPainter::shear](qpainter.html#shear)() function. All the transformation operations operate on [QPainter](qpainter.html)\'s transformation matrix that you can retrieve using the [QPainter::worldTransform](qpainter.html#worldTransform)() function. A matrix transforms a point in the plane to another point.

If you need the same transformations over and over, you can also use [QTransform](qtransform.html) objects and the [QPainter::worldTransform](qpainter.html#worldTransform)() and [QPainter::setWorldTransform](qpainter.html#setWorldTransform)() functions. You can at any time save the [QPainter](qpainter.html)\'s transformation matrix by calling the [QPainter::save](qpainter.html#save)() function which saves the matrix on an internal stack. The [QPainter::restore](qpainter.html#restore)() function pops it back.

One frequent need for the transformation matrix is when reusing the same drawing code on a variety of paint devices. Without transformations, the results are tightly bound to the resolution of the paint device. Printers have high resolution, e.g. 600 dots per inch, whereas screens often have between 72 and 100 dots per inch.

## Window-Viewport Conversion

When drawing with [QPainter](qpainter.html), we specify points using logical coordinates which then are converted into the physical coordinates of the paint device.

The mapping of the logical coordinates to the physical coordinates are handled by [QPainter](qpainter.html)\'s world transformation [worldTransform](qpainter.html#worldTransform)() (described in the [Transformations](coordsys.html#transformations) section), and [QPainter](qpainter.html)\'s [viewport](qpainter.html#viewport)() and [window](qpainter.html#window)(). The viewport represents the physical coordinates specifying an arbitrary rectangle. The \"window\" describes the same rectangle in logical coordinates. By default the logical and physical coordinate systems coincide, and are equivalent to the paint device\'s rectangle.

Using window-viewport conversion you can make the logical coordinate system fit your preferences. The mechanism can also be used to make the drawing code independent of the paint device. You can, for example, make the logical coordinates extend from (-50, -50) to (50, 50) with (0, 0) in the center by calling the [QPainter::setWindow](qpainter.html#setWindow)() function:

::: pre
``` {.cpp .prettyprint translate="no"}
QPainter painter(this);
painter.setWindow(QRect(-50, -50, 100, 100));
```

Now, the logical coordinates (-50,-50) correspond to the paint device\'s physical coordinates (0, 0). Independent of the paint device, your painting code will always operate on the specified logical coordinates.

By setting the \"window\" or viewport rectangle, you perform a linear transformation of the coordinates. Note that each corner of the \"window\" maps to the corresponding corner of the viewport, and vice versa. For that reason it normally is a good idea to let the viewport and \"window\" maintain the same aspect ratio to prevent deformation:

::: pre
``` {.cpp .prettyprint translate="no"}
int side = qMin(width(), height());
int x = (width() - side / 2);
int y = (height() - side / 2);

painter.setViewport(x, y, side, side);
```

If we make the logical coordinate system a square, we should also make the viewport a square using the [QPainter::setViewport](qpainter.html#setViewport)() function. In the example above we make it equivalent to the largest square that fit into the paint device\'s rectangle. By taking the paint device\'s size into consideration when setting the window or viewport, it is possible to keep the drawing code independent of the paint device.

Note that the window-viewport conversion is only a linear transformation, i.e. it does not perform clipping. This means that if you paint outside the currently set \"window\", your painting is still transformed to the viewport using the same linear algebraic approach.

![Illustration showing how coordinates are mapped using viewport, \"window\" and transformation matrix](images/coordinatesystem-transformations.png)

The viewport, \"window\" and transformation matrix determine how logical [QPainter](qpainter.html) coordinates map to the paint device\'s physical coordinates. By default the world transformation matrix is the identity matrix, and the \"window\" and viewport settings are equivalent to the paint device\'s settings, i.e. the world, \"window\" and device coordinate systems are equivalent, but as we have seen, the systems can be manipulated using transformation operations and window-viewport conversion. The illustration above describes the process.
