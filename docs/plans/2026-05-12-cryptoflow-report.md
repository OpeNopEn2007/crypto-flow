# CryptoFlow -- 交互式密码学原理可视化演示系统

## 项目概述

CryptoFlow 是一个面向教学的交互式密码学原理可视化演示系统，作为南开大学密码学课程期末作业开发。项目旨在通过直观的动画演示，帮助学生理解经典密码学算法的工作原理。

系统当前支持两种算法的可视化：凯撒密码（对称加密）和 RSA 算法（非对称加密）。用户可以通过图形界面选择算法、调整参数、控制动画速度，也可以通过命令行参数自动启动演示。

---

## 技术实现

### 技术栈

| 组件 | 选型 |
|------|------|
| 语言标准 | C++20 |
| GUI 框架 | Qt 6 (Widgets) |
| 构建系统 | CMake 3.20 |
| 动画引擎 | QGraphicsView/QGraphicsScene + QTimer + QEasingCurve |
| 主题样式 | QSS 深色科技风（GitHub Dark 色系） |

### 架构设计

项目采用分层架构，从底层算法到顶层 UI 逐层封装：

```
crypto/   -- 纯算法层（CaesarCipher、RSAEngine），无 UI 依赖
scenes/   -- 动画场景层（CaesarScene、RSAScene），继承 QGraphicsScene
controlpanel -- UI 控制层（ControlPanel），算法选择、参数输入、速度调节
mainwindow   -- 组装层，用 QSplitter 布局，连接场景与控制面板
```

层间通信全部通过 Qt 信号槽机制完成，且严格使用新式语法（`&Class::method`），禁止旧式 `SIGNAL/SLOT` 宏。这种设计保证了模块间的松耦合，算法层可以独立测试和复用。

### 调试与自动化

- **双日志系统**：通过自定义 `messageHandler` 同时输出到 stderr 和 `cryptoflow.log` 文件，方便调试和回溯。
- **自动截图**：动画完成后自动保存 `caesar_screenshot.png` / `rsa_screenshot.png`。
- **CLI 参数**：支持 `--caesar "HELLO" --shift 3` 和 `--rsa` 参数，可无人值守自动运行演示。

## 凯撒密码可视化

### 转盘设计

凯撒密码的可视化采用双圈转盘结构：

- **外圈**（半径 180px，青色 #00C8FF）：固定不动，代表明文字母表 A-Z，字母沿圆周均匀分布（每 13.85 度一个）。
- **内圈**（半径 120px，橙色 #FF6432）：可旋转，代表密文字母表 A-Z，初始状态与外圈对齐。

两个圈均带有 `QGraphicsDropShadowEffect` 霓虹发光效果，外圈模糊半径 25，内圈 18，并叠加正弦脉冲动画（每 50ms 更新一次），产生持续呼吸感。

### 动画流程

动画分为四个阶段，按顺序执行：

1. **旋转阶段**：内圈逆时针旋转，旋转角度为 `-shift * (360 / 26)` 度。采用 OutCubic 缓动曲线（先快后慢），共 40 步，每步 35ms。逆时针旋转使得密文字母表的移位方向与加密逻辑一致。

2. **逐字母高亮阶段**：对输入文本中的每个字母，同时高亮外圈的明文字母和内圈对应的密文字母。高亮使用黄色（#FFFF00）发光效果，模糊半径 30。每次高亮持续 `animSpeed_` 毫秒，然后清除进入下一轮。

3. **连接线动画**：在高亮的同时，从外圈字母位置向内圈字母位置绘制虚线连接线。连线动画共 12 步，每步 25ms，前端带有一个黄色光点，模拟数据流动效果。

4. **打字机效果**：所有字母处理完成后，结果文本以逐字显示的方式出现在场景中央，字体为 Menlo 22px 加粗，绿色（#00FF96）发光效果，每字间隔 80ms。

### 技术难点

**内圈字母旋转倾斜问题**：最初尝试使用 `QGraphicsItemGroup` 将内圈圆环和字母编组后整体旋转，但发现旋转会导致字母本身也随之倾斜，影响可读性。解决方案是将字母作为独立的 `QGraphicsTextItem`，在 `setInnerRingRotation()` 中通过 cos/sin 手动重算每个字母的位置坐标，仅旋转圆环本身：

```cpp
void CaesarScene::setInnerRingRotation(double deg) {
    innerRing_->setRotation(deg);
    for (int i = 0; i < 26; i++) {
        double angle = (i * 360.0 / 26 - 90 + deg) * M_PI / 180;
        double x = CENTER_X + RADIUS_INNER * 0.8 * cos(angle) - 8;
        double y = CENTER_Y + RADIUS_INNER * 0.8 * sin(angle) - 10;
        innerLetters_[i]->setPos(x, y);
    }
}
```

**负方向旋转判断**：`targetRotation_` 为负值（逆时针），在动画插值时需要直接使用负值乘以缓动进度，而非取绝对值后判断方向，否则会导致旋转方向反转。

## RSA 可视化

### 密钥生成动画

RSA 密钥生成采用流程图式卡片展示，共 7 步，每步以卡片形式依次出现：

| 步骤 | 内容 | 卡片颜色 |
|------|------|---------|
| 1 | 选择素数 p, q | 蓝色（#00C8FF） |
| 2 | 计算 n = p x q | 蓝色 |
| 3 | 计算欧拉函数 phi(n) = (p-1)(q-1) | 蓝色 |
| 4 | 选择公钥指数 e，满足 gcd(e, phi(n)) = 1 | 蓝色 |
| 5 | 计算私钥指数 d = e^-1 (mod phi(n)) | 蓝色 |
| 6 | 公钥 (e, n) | 绿色（#00FF96） |
| 7 | 私钥 (d, n) | 金色（#FFB400） |

每张卡片包含：左侧彩色竖条、步骤编号、描述文本、公式、计算结果值。卡片之间用带箭头的连接线衔接，形成清晰的流程感。动画速度由 `animSpeed_` 控制（默认 800ms/步）。

### 加密/解密动画

加密和解密各有独立的三步动画流程：

- **加密**：展示明文 M -> 公钥 (e, n) -> 加密公式 C = M^e mod n
- **解密**：展示密文 C -> 私钥 (d, n) -> 解密公式 M = C^d mod n

### 视觉编码

- 蓝色系 = 计算步骤（纯数学运算）
- 绿色 = 公钥相关（公钥指数 e、加密结果）
- 金色 = 私钥相关（私钥指数 d）
- 橙色 = 密文数据

### 底层算法

`RSAEngine` 类实现了完整的 RSA 算法逻辑：
- `isPrime()`：试除法素数判定
- `gcd()`：欧几里得算法
- `modPow()`：快速幂取模（二进制展开法）
- `modInverse()`：扩展欧几里得算法求模逆元
- 公钥指数 e 默认选取 65537（常用值），若与 phi(n) 不互素则递增 2 直到满足条件

## 界面设计

### 布局

主窗口采用 `QSplitter` 水平分割，左侧为 `QGraphicsView`（占 3/4 宽度）用于动画展示，右侧为 `ControlPanel`（固定 280px）用于参数控制。

### 主题

整体采用 GitHub Dark 风格的 QSS 深色主题：

- 主背景色：`#0d1117`
- 面板背景色：`#161b22`
- 画布背景色：`#1a1a2e`
- 主强调色：`#58a6ff`（霓虹蓝）
- 辅助强调色：`#00FF96`（霓虹绿）
- 输入框字体：Menlo 等宽字体

开始按钮为绿色（`#238636`），重置按钮为红色（`#da3633`），与 GitHub 界面风格一致。

### 控件

控制面板通过 `QStackedWidget` 实现算法切换：

- **凯撒密码面板**：文本输入框（默认 "HELLO"）、移位量微调框（1-25，默认 3）
- **RSA 面板**：素数 p/q 微调框（2-997）、模式选择（生成密钥/加密/解密）、数值输入、公钥/私钥/模数显示
- **公共控件**：算法选择 ComboBox、速度滑块（100-1500ms）、开始/重置按钮

## 开发过程

项目经历了从基础骨架到完整动画的多轮迭代：

1. **骨架搭建**：先实现 MainWindow + ControlPanel 的基本布局，确认 QSplitter 和 QStackedWidget 的工作方式。
2. **凯撒算法层**：实现 CaesarCipher 的加密/解密逻辑，为动画层提供数据支撑。
3. **凯撒转盘动画**：先绘制静态转盘，再逐步添加旋转、高亮、连接线、打字机效果。
4. **RSA 算法层**：实现 RSAEngine，包含密钥生成、加密、解密和步骤生成。
5. **RSA 场景动画**：实现卡片式流程图展示，添加颜色编码。
6. **视觉调优**：添加霓虹发光效果、脉冲动画、连接线光点、打字机效果等细节。
7. **CLI 和截图**：添加命令行参数解析和动画完成自动截图功能。

### 关键问题解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 内圈字母旋转倾斜 | QGraphicsItemGroup 整体旋转带动子元素旋转 | 改用独立 QGraphicsTextItem + cos/sin 手动定位 |
| 旋转方向与加密逻辑不一致 | 默认顺时针旋转导致字母映射反向 | targetRotation_ 取负值实现逆时针旋转 |
| 打字机效果与高亮逻辑冲突 | 两者共用 highlightTimer_ 导致状态混乱 | 引入 typewriterActive_ 标志位，分离两个阶段的状态机 |
